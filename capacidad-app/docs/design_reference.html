Cuándo usar cada botón

btn-primary — acción principal de la tarjeta (Calcular, Verificar, Importar)

btn-secondary — acciones secundarias (Visualizar, Exportar CSV, Ver errores)

btn-ghost — acciones de contexto (cerrar modal, cambiar carpeta, toggles)

btn-danger — acciones destructivas (eliminar regla, borrar datos)
Modificadores de tamaño

btn-sm — botones dentro de tarjetas, en tablas

(normal) — botones de formulario principales

btn-lg — CTAs principales de página

disabled — aplicar atributo HTML disabled, no clase CSS

15. Modo Claro / Modo Oscuro
Cómo funciona
El tema se controla con el atributo data-theme="light" o data-theme="dark" en document.documentElement.

El componente ThemeToggle (pie del sidebar) alterna entre temas y persiste la preferencia en localStorage con la clave rcCapacityTheme.

Regla crítica: nunca usar colores hexadecimales hardcodeados en JSX. Siempre usar variables CSS como var(--card-success-bg). Esto garantiza que el cambio de tema funcione automáticamente sin código JS adicional.

Los colores semánticos (--success, --error, --warning) son idénticos en ambos temas. Solo cambian los fondos estructurales, textos y bordes.
